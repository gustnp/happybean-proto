# 해피빈 캠페인 프로토타입 시스템

해피빈 커스텀 캠페인 페이지의 기획서·시안·제작 스펙을 하나의 링크로 제공하는 정적 사이트.
빌드 과정 없음. `index.html` + `campaigns/*.js` 데이터 파일이 전부다.

## 구조

- `index.html` — 뷰어. 캠페인 데이터를 읽어 렌더링. 상단 토글로 3개 모드 전환:
  - **시안**: 클라이언트용 완성 페이지 뷰
  - **기획 문서**: 블록별 기획 의도(plan) + 캠페인 메타(목표/KPI/일정) 표시 — PPT 기획서 대체
  - **제작 스펙**: 블록별 필요 에셋·인터랙션 정의(spec) 표시 — 디자이너/개발자용
- `campaigns/캠페인아이디.js` — 캠페인 1개 = 파일 1개. 모든 텍스트/테마/미션 구성이 여기 있음
- `campaigns/_template.js` — 새 캠페인 시작점
- 접속: `index.html?c=캠페인아이디` (파라미터 없으면 첫 캠페인)

## 페이지 구조 규칙 (해피빈 커스텀 페이지 문법)

고정 골격(순서 불변): GNB → 히어로 → **[미션 슬롯 N개]** → SNS 밴드 → 기부금 안내 → 공유 밴드 → 댓글 참여 → 푸터

- GNB, 푸터, 댓글 컴포넌트, 기부금/공유 밴드 레이아웃은 해피빈 공통 — 구조 수정 금지, 텍스트만 교체
- 미션 슬롯이 캠페인별 크리에이티브 영역. 프리셋 타입(quiz/story/channels/video/cards/cta) 또는 custom 사용
- custom 미션은 반드시 spec.interaction에 동작 정의(트리거 → 결과 → 콩 지급 조건)를 서술할 것
- 콩 지급이 있는 미션은 bean 필드 필수 ("미션 완료 시 콩 1개")
- 플로팅 위젯: 참여수형(counter) 또는 스탬프 트래커형(stamps) 중 캠페인 성격에 맞게 선택
- 타이틀·카피는 가운데 정렬이 기본. 줄바꿈은 \n

## 새 캠페인 추가 절차

1. `campaigns/_template.js`를 `campaigns/<영문아이디>.js`로 복사
2. `window.CAMPAIGNS["<영문아이디>"]`로 키 변경 (파일명과 일치)
3. 사용자가 준 카피·구성·클라이언트 정보로 내용 채우기
   - plan 필드: 각 블록의 기획 의도를 1~2문장으로 (비어 있으면 기획 문서 모드가 빈약해짐 — 반드시 작성)
   - spec 필드: 필요 에셋을 구체적으로 (규격 포함), 인터랙션은 콩 지급 조건까지
   - theme: 클라이언트 브랜드 컬러 기준으로 primary/heroFrom/heroTo/band1/band2 지정
4. `index.html`의 캠페인 로드 구간에 `<script src="campaigns/<영문아이디>.js"></script>` 한 줄 추가
5. 로컬 확인 후 커밋·푸시

## 커밋/배포 규칙

- 커밋 메시지: `campaign: <캠페인명> 시안 v<버전>` / 수정은 `fix:`, 시스템 변경은 `system:`
- push 후 배포 URL과 `?c=` 파라미터가 붙은 최종 공유 링크를 사용자에게 출력할 것
- 배포: 레포는 **비공개**. GitHub Pages(Pro) 또는 Vercel 연동 기준. 정적 파일이므로 push만으로 배포 완료
- 이미지 에셋은 `assets/<캠페인아이디>/`에 저장하고 캠페인 파일에서 상대경로 참조

## 주의

- 실제 파트너사명·미공개 캠페인 정보가 들어가므로 레포를 공개로 전환하지 말 것
- 댓글 수, 참여 수 등 숫자는 시안용 더미임을 인지 (실서비스 수치 아님)
- 뷰어(index.html)의 렌더러를 수정할 때는 기존 캠페인 파일들이 깨지지 않는지 전 캠페인 확인
